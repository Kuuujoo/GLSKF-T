# GLSKF-tSVD 实验重做计划

## 总体安排
按阶段推进，每阶段结束后停下来汇报，并更新 `STATUS.md`。

阶段顺序如下。

1. 图像优先
2. 视频五方法实验
3. 随机低秩张量数据实验
4. 修改 `论文编写/20260225.tex`

每阶段记录文件改动、运行命令、成功结果、失败错误、当前最好指标和下一步建议。

## 第一阶段 图像实验
目标是先解决 `Airplane` 在 90% 缺失率下的结果。

要求如下。

- 固定随机种子 `920`
- 使用观测位置全局更新版本
- 修改入口为 `图像实验/GLSKF_T.m`
- 修改算法文件为 `图像实验/GLSKF-T/GLSKF_tSVD.m`
- 不覆盖 `*_ori.m`
- 不切回 ori 版本
- 重点调 `K0` 和 `maxiter`
- 验收标准为 PSNR 大于 29

参数搜索调整为小迭代版本。

第一轮粗搜。

- `K0 = [2, 3, 4, 5, 6, 8]`
- `maxiter = [20, 25, 30, 35, 40]`
- `rho = [3, 5, 7, 10, 15, 20, 30]`
- `gamma = [3, 5, 7, 10, 15, 20, 30]`
- `rg = [5, 7, 10]`

判断规则。

- 如果全局部分后 PSNR 一直下降，优先判定为 `K0` 偏大
- 遇到这种情况，下一轮优先降低 `K0`
- 不使用 `maxiter = 50` 或 `80`
- 若 PSNR 仍未超过 29，停止并汇报，不擅自换算法口径

输出保存到 `图像实验/results/GLSKF-T/airplane90_tuning/`。

必须保存。

- 每组参数汇总 CSV
- 最优结果 MAT
- 每轮 PSNR、MSE、RMSE、时间 CSV
- 原图、观测图、恢复图
- 当前最好参数说明

## 第二阶段 视频实验
视频实验使用 YUV，不使用 MP4。

数据如下。

- `视频实验/处理后视频/news_qcif_gray.yuv`
- `视频实验/处理后视频/akiyo_qcif_gray.yuv`
- 尺寸 `144 x 176`
- 缺失率 `80%`、`90%`、`95%`
- 使用脚本里已有的掩码生成逻辑和随机种子设置

方法如下。

- GLSKF
- GLSKF-T
- SPC
- HaLRTC
- GLTC-TNN

GLSKF 和 GLSKF-T 以现有脚本为基础，主要负责监控运行和补齐结果整理。  
SPC、HaLRTC、GLTC-TNN 需要改成可重复运行、可保存日志、可画统一曲线的版本。

掩码规则如下。

- 不新增统一 Omega 生成脚本
- 不强制改成外部共享 Omega 文件
- 保留各方法脚本里已有的掩码生成部分和种子设置
- 每次运行保存实际使用的 Omega，方便后续核对

统一输出字段如下。

- dataset
- method
- variant
- missing_rate
- seed
- iteration
- elapsed_time_seconds
- MSE
- RMSE
- PSNR
- SSIM
- parameter settings
- convergence status

视频输出保存到 `视频实验/results/`。

每个组合保存。

- per-iteration CSV
- final metrics CSV
- MAT 或 NPZ 结果文件
- 实际使用的 Omega
- 关键帧原图、观测图、恢复图
- 每个数据集和缺失率的 MSE 时间曲线
- 每个数据集和缺失率的 RMSE 时间曲线
- 五种方法汇总表格

## 第三阶段 随机低秩张量实验
只做中等规模正式实验。

张量设置如下。

- 尺寸 `120 x 120 x 30`
- tubal rank 为 `8`
- 随机重复 3 次
- 种子为 `920`、`921`、`922`
- 缺失率为 `80%`、`90%`、`95%`

生成方式如下。

- 先固定对角形式的 `S`
- 生成随机 `U` 和 `V`
- 用 t-product 得到完整张量 `X = U * S * V^T`
- 将 `X` 归一化到 `[0, 1]`
- 添加随机缺失
- 保存每次实验实际使用的数据和掩码

结果保存到 `results/synthetic_lowrank/`。

输出如下。

- 每次重复的 MSE 和 RMSE 时间曲线
- 三次重复的均值和标准差表格
- 每个缺失率的五方法 MSE 时间曲线
- 每个缺失率的五方法 RMSE 时间曲线
- 生成张量和 Omega 的 MAT 文件
